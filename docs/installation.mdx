# Get Started

## Choose how you want to use everyrow

### [Web App](https://everyrow.io/app)

Only setup is signing in with Google at [everyrow.io/app](https://everyrow.io/app).

### [Claude.ai](/claude-ai)

Add a connector in Settings. Works in Claude Desktop too.

### [Claude Cowork](/claude-cowork)

Add the connector in Settings.

### [Claude Code](/claude-code)

One command: `claude mcp add everyrow --scope project --transport http https://mcp.everyrow.io/mcp`

### [Python SDK](/getting-started)

For developers writing Python. Requires `pip install everyrow` and an [API key](https://everyrow.io/api-key).

## Other AI agents

These integrations use the local MCP server with an API key. Get one at [everyrow.io/api-key](https://everyrow.io/api-key).

<InstallationTabs>

<TabContent agent="codex" type="skills">

Ask Codex in natural language:

```
>>> codex
>>> Install the skill from github.com/futuresearch/everyrow-sdk found under skills/everyrow-sdk
```

Or install manually:

```bash
python ~/.codex/skills/.system/skill-installer/scripts/install-skill-from-github.py \
  --repo futuresearch/everyrow-sdk --path skills/everyrow-sdk
```

Restart Codex to pick up the new skill.

**Important:** be sure to supply your API key when launching Codex:

```bash
export EVERYROW_API_KEY=sk-cho...
codex
```

[Official Docs](https://developers.openai.com/codex/skills#install-new-skills)

</TabContent>

<TabContent agent="codex" type="mcp">

Add everyrow to your Codex MCP config (requires [uv](https://docs.astral.sh/uv/)):

```toml
[mcp_servers.everyrow]
command = "uvx"
args = ["everyrow-mcp"]

[mcp_servers.everyrow.env]
EVERYROW_API_KEY = "${EVERYROW_API_KEY}"
```

Or use the CLI:

```bash
codex mcp add everyrow --env EVERYROW_API_KEY=${EVERYROW_API_KEY} -- uvx everyrow-mcp
```

Config file location:
- **User scope:** `~/.codex/config.toml`
- **Project scope:** `.codex/config.toml`

**Important:** either insert your API key when creating the config file, or supply the key when launching Codex:

```bash
export EVERYROW_API_KEY=sk-cho...
codex
```

[Official Docs](https://developers.openai.com/codex/mcp/)

</TabContent>

<TabContent agent="gemini" type="skills">

Install the skill directly from GitHub:

```bash
gemini skills install https://github.com/futuresearch/everyrow-sdk --path skills/everyrow-sdk
```

Use `--scope workspace` to install to `.gemini/skills/` (project-level) instead of `~/.gemini/skills/` (user-level).

Then enable skills within the CLI:

```
/settings > Preview Features > Enable
/settings > Agent Skills > Enable
/skills enable everyrow-sdk
/skills reload
```

[Official Docs](https://geminicli.com/docs/cli/skills/)

</TabContent>

<TabContent agent="gemini" type="mcp">

Add via CLI (requires [uv](https://docs.astral.sh/uv/)):

```bash
gemini mcp add -e EVERYROW_API_KEY=$EVERYROW_API_KEY everyrow uvx everyrow-mcp
```

Or add manually to `~/.gemini/settings.json` (user) or `.gemini/settings.json` (project):

```json
{
  "mcpServers": {
    "everyrow": {
      "command": "uvx",
      "args": ["everyrow-mcp"],
      "env": {
        "EVERYROW_API_KEY": "$EVERYROW_API_KEY"
      }
    }
  }
}
```

[Official Docs](https://geminicli.com/docs/tools/mcp-server/)

</TabContent>

<TabContent agent="gemini" type="plugin">

This installs both the MCP server and skills together.

```bash
gemini extensions install https://github.com/futuresearch/everyrow-sdk --ref main
```

When prompted with `Would you like to attempt to install via "git clone" instead?`, type `Y` to confirm.

Then enable the extension:

```bash
gemini extensions enable everyrow [--scope <user or workspace>]
```

To use skills, enable them within the CLI:

```
/settings > Preview Features > Enable
/settings > Agent Skills > Enable
/skills enable everyrow-sdk
/skills reload
/model > Manual > gemini-3-pro-preview
```

[Official Docs](https://geminicli.com/docs/extensions/#installing-an-extension)

</TabContent>

## Dependencies

The MCP server requires [**uv**](https://docs.astral.sh/uv/) (if using `uvx`) or [**pip**](https://pip.pypa.io/en/stable/) (if installed directly). The Python SDK requires **Python 3.12+**.

For the optional terminal progress bar, see the [jq dependency](/docs/progress-monitoring#status-line-progress-bar) in the progress monitoring guide.
